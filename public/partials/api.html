API

Zenobase has an open API that developers can use to create custom integrations with 3rd party data sources, or access and update data on behalf of users. For a complete example that shows how to authenticate with and post data to Zenobase, look at this Android app, or find additional tips and examples on our blog.

Authentication

Requests for protected resources must include an OAuth 2 Authorization header:

Authorization: Bearer <access_token>

Access tokens are JWTs issued through the Auth0 hosted login flow used by the Zenobase web app, and expire according to the Auth0 session policy. Sign in to the web app and copy your current token from the settings page.

Buckets

Buckets are containers for events, but also include permissions and dashboard settings.

List buckets

GET /users/<user_id>/buckets/?offset=<offset>&limit=<limit>&include_archived=<include_archived>
Host: api.zenobase.com
Authorization: Bearer <access_token>
user_id
The user whose buckets we want to retrieve.
offset
How many buckets to skip. Defaults to 0.
limit
The number of buckets to return, up to 100. Defaults to 10.
include_archived
Whether to include archived buckets. Defaults to false.
Example
curl -i 'https://api.zenobase.com/users/args4iqh2c/buckets/?limit=10' \
  -H 'Authorization: Bearer e7975e8bb6eabc3ef567'
HTTP/1.1 200 OK
Content-Type: application/json
{
  "total" : 1,
  "buckets" : [
    {
      "@id" : "jl2rg0o36e",
      "version" : 1,
      "label" : "My Data",
      "created" : "2012-10-18T20:59:36.295Z",
      "size" : 4,
      "roles" : [
        { "principal" : "args4iqh2c", "role" : "owner" }
      ]
    }
  ]
}

Create a bucket

POST /buckets/
Content-Type: application/json
Host: api.zenobase.com
Authorization: Bearer <access_token>
{
  "label" : <label>,
  "description" : <description>,
  "archived" : <archived>,
  "aliases" : [
    { "@id" : <bucket_id> },
    { "@id" : <bucket_id> },
    ...
}
label
A human-readable name for the bucket. Required, 1-20 characters from a-zA-Z0-9-_ .
description
One or two sentences describing this bucket. Optional.
archived
Whether this bucket is archived; archived buckets are not listed by default. Optional.
aliases
Optional list of existing buckets; creates a virtual bucket that serves as a read-only view.

curl -i -X POST 'https://api.zenobase.com/buckets/' \
  -H 'Content-Type: application/json' \

  -d '{
    "label" : "Test Bucket",

Update a bucket

Update a bucket's metadata, i.e. its label, description, roles and widgets.

PUT /buckets/<bucket_id>
Content-Type: application/json
Host: api.zenobase.com
Authorization: Bearer <access_token>
{
  "label" : <label>,
  "description" : <description>,
  "archived" : <archived>,
  "roles" : [
    {
      "principal" : <user_id>,
      "role" : <'owner'|'viewer'>
    },
    ...
  ],
  "widgets" : [
    ...
  ]
}
bucket_id
The bucket to update.
Example:
curl -i -X PUT 'https://api.zenobase.com/buckets/v7jg3l5ibc' \
  -H 'Content-Type: application/json' \
  -H 'Authorization: Bearer e7975e8bb6eabc3ef567 \
  -d '{
    "label" : "Updated Bucket",
    "description" : "This bucket is archived, but still public!",
    "archived" : true,
    "roles" : [
      { "principal" : "args4iqh2c", "role" : "owner" },
      { "principal" : "*", "role" : "viewer" }
    ]
  }'
HTTP/1.1 204 No Content
Content-Type: application/json
X-Command-ID: 4o6iqmpnrr

Delete a bucket

Delete a bucket including all its events.

DELETE /buckets/<bucket_id>
Host: api.zenobase.com
Authorization: Bearer <access_token>
bucket_id
The bucket to delete; must not be aliased from a virtual bucket.
Example:
curl -i -X DELETE 'https://api.zenobase.com/buckets/v7jg3l5ibc' \
  -H 'Authorization: Bearer e7975e8bb6eabc3ef567'
HTTP/1.1 204 No Content
X-Command-ID: p3atfetvs2

Events

Events contain the actual data being tracked.

List events

GET /buckets/<bucket_id>/?offset=<offset>&limit=<limit>&order=<order>&facet=<facet_expression>&q=<constraint_expression>
Host: api.zenobase.com
Authorization: Bearer <access_token>
bucket_id
The bucket from which to retrieve events.
offset
The offset of the first event. Defaults to 0.
limit
How many events to return. If no limit and no facets are specified, a list of all matching events is returned.
order
The field to sort on, descending if prefixed with '-'. Defaults to -timestamp.
facet_expression
See facets. More than one facet parameter can be given.
constraint_expression
See constraints. Optional. If more than one constraint parameter is present, all constraints must be fulfilled.
Example:
curl -i 'https://api.zenobase.com/buckets/v7jg3l5ibc/?offset=0&limit=10&facet=id:tags,type:count&q=tag:lunch' \
  -H 'Authorization: Bearer e7975e8bb6eabc3ef567'
HTTP/1.1 200 OK
Content-Type: application/json
{
  "total" : 42,
  "events": [
    {
      "timestamp" : "2012-10-24T13:10:00.000-07:00",
      "tag" : [ "lunch", "thai" ],
      "author" : "7ghkds6jar",
      ...
    },
    ...
  ],
  "tags": [
    { "label" : "lunch", "count" : 10 },
    { "label" : "thai", "count" : 1 },
    ...
  ]
}

Ranges

The begin or end of a range expressions can be open, inclusive or exclusive:

[a..b]
values not smaller than a and not larger than b
[a..b)
values not smaller than a but smaller than b
(a..b]
values larger than a but not larger than b
(a..b)
values larger than a and smaller than b
[a..*)
values not smaller than a
(*..b]
values not larger than b

Pseudo-Fields

_timestamp
The last modification time of an item. For example, _timestamp:[2013TZ..*) matches items that have been modified since the beginning of 2013.
<field>$min
The lowest value in a field with multiple values. Currently only supported for timestamp fields.
<field>$max
The highest value in a field with multiple values. Currently only supported for timestamp fields.

Facets

Facets can be used to specify what data to return e.g. when listing events.

id:<id>,type:<type>,<param:value>,...
id
A short string, used to identify the facet's data in the result.
type
The facet type, see below.
param:value
Type-specific parameters for the facet, see below.
 

Count Facet

field:<'tag'|'author'>
The field to aggregate on. Required.
offset:<integer>
The offset of the first result. Defaults to 0.
limit:<integer>
How many results to return. Defaults to 10.
order:<'-'><'term'|'count'>
Order results by term or by count, descending if prefixed with '-'. Defaults to -count.
filter:<constraint>
Optional constraint.
Example:
id:xyz,type:count,field:tag,order:term
"xyz" : [
  { "label" : "breakfast", "count" : 19 }
  { "label" : "dinner", "count" : 31 },
  { "label" : "lunch", "count" : 22 },
]
 

Timeline Facet

field:<String>
The field containing the values. Required.
unit:<String>
The unit of the value. Required for fields with units.
interval:<'year'|'month'|'day'|'hour'|'minute'>
The interval at which values are aggregated. Defaults to month.
timezone:<Z>
Optional timezone offset, e.g.+00:00. If not set, values are aggregated using their local time.
filter:<constraint>
Optional constraint.
key_field:<'timestamp'|'timestamp$min'|'timestamp$max'>
How to handle events with multiple timestamps. Defaults to timestamp, i.e. aggregate on each timestamp, rather than on the highest or lowest timestamp.
Example:
id:xyz,type:timeline,field:count,filter:tag:steps
"xyz" : [
  { "label" : "2013-01", "count" : 31, "avg" : 9832, "min" : 2559, "max" : 24600, "sum" : 304792 },
  { "label" : "2013-02", "count" : 28, "avg" : 12225.93, "min" : 4258, "max" : 19817, "sum" : 342326 },
  { "label" : "2013-03", "count" : 31, "avg" : 24134, "min" : 4512, "max" : 24134, "sum" : 317220 },
  ...
]

Controls the clustering, from 0.0 (show each point) to 1.0 (one big cluster). Defaults to 0.2.
Example:
id:xyz,type:map,factor:0.4
"xyz" : [
  { "count" : 1, "lat" : -2.8, "lon" : -40.48 },
  { "count" : 14, "lat" : 14.895, "lat_max" : 16.512, "lat_min" : -3.784, "lon" : -61.823, "lon_max" : -61.048, "lon_min" : -64.950 },
  ...
]
 

Heatmap Facet

precision:<integer>
Determines the grid size, from 1 (~5,000km) to 10 (~1m). Defaults to 8 (~30m).
value_field:<String>
An optional, numeric field containing values to sum up.
unit:<String>
The unit of the values. Required for fields with units.
filter:<constraint>
Optional constraint.
Example:
id:xyz,type:heatmap,precision:4,field:duration
"xyz" : [
  { "lat" : -2.8, "lon" : -40.48, "count" : 1, "sum" : 60000 },
  { "lat" : 14.895, "lon" : -61.823, "count" : 3, "sum" : 180000 },
  ...
]

Scoreboard Facet

key_field:<'author'|'tag'>
The field to aggregate on. Required.
value_field:<String>
The field containing the values. Required.
unit:<String>
The unit of the value. Required for fields with units.
limit:<integer>
How many results to return. Defaults to 10.
order:<'-'><'term'|'count'|'min'|'max'|'sum'|'avg'>
Order results by term or on a statistic, descending if prefixed with '-'. Defaults to -sum.
filter:<constraint>
Optional constraint.
Example:
id:xyz,type:scoreboard,key_field:author,value_field:steps,order:min
"xyz" : [
  { "label" : "dch4v3salm", "count" : 364, "min" : 1640, "max" : 38215, "sum" : 3899768, "avg" : 10713.65 },
  ...
]
 

Scatter Plot Facet

field_x:<String>
The field for the x series. Required.
unit_x:<String>
The unit for the x series. Required for fields with units.
statistic_x:<'count'|'avg'|'min'|'max'|'sum'>
The aggregate function for the x series. Defaults to avg.
filter_x:<constraint>
Optional constraint for the x series.
field_y:<String>
The field for the y series. Required.
unit_y:<String>
The unit for the y series. Required for fields with units.
statistic_y:<'count'|'avg'|'min'|'max'|'sum'>
The aggregate function for the y series. Defaults to avg.
filter_y:<constraint>
Optional constraint for the y series.
interval:<'year'|'month'|'week'|'day'|'hour'|'minute'>
The interval at which values are aggregated. Defaults to day.
timezone:<Z>
The timezone offset to use. If not specified, local times are used.
lag:<Integer>
The number of time units (see interval) by which to offset the y series values from the x series values. Defaults to 0.
key_field:<'timestamp'|'timestamp$min'|'timestamp$max'>
How to handle events with multiple timestamps. Defaults to timestamp, i.e. aggregate on each timestamp, rather than on the highest or lowest timestamp.
Example:
id:xyz,type:scatterplot,field_x:duration,field_y:duration,statistic:sum,interval:day,lag:1
"xyz" : [
  [ 2.826E+7, 2.592E+7 ],
  [ 2.898E+7, 2.826E+7 ],
  [ 3.126E+7, 2.898E+7 ],
  ...
]
 

Gantt Facet

field:<String>
The field containing the values. Required.
limit:<integer>
How many results to return. Defaults to 10.
order:<-><'term'|'count'|'min'|'max'>
Order results by term, count, first (min) or last (max) occurence, descending if prefixed with '-'. Defaults to -max.
timezone:<Z>
The timezone offset to use. Defaults to +00:00
key_field:<'timestamp'|'timestamp$min'|'timestamp$max'>
How to handle events with multiple timestamps. Defaults to timestamp, i.e. aggregate on each timestamp, rather than on the highest or lowest timestamp.
filter:<constraint>
Optional constraint.
Example:
id:xyz,type:gantt,field:tag,order:-max,timezone:-08:00
"xyz" : [
  { "label" : "dinner", "count" : 22, "first" : "2013-01-01T18:23:00.000-08:00", "last":"2013-11-07T19:45:00.000-08:00" },
  { "label" : "lunch", "count" : 31, "first" : "2013-01-01T13:23:00.000-08:00", "last":"2013-11-07T12:05:00.000-08:00" },
  ...
]

Polar Facet

key_field:<String>
A field containing a timestamp. Defaults to timestamp.
value_field:<String>
A field containing the values to be aggregated. Optional; if not set, only a count is returned for each interval.
unit:<String>
The unit of the values. Required for fields with units.
interval:<'hour_of_day'|'day_of_week'|'day_of_month'|'day_of_year'|'month_of_year'>
The (local, timezone-independent) interval to aggregate on. Required.
filter:<constraint>
Optional constraint.
key_field:<'timestamp'|'timestamp$min'|'timestamp$max'>
How to handle events with multiple timestamps. Defaults to timestamp, i.e. aggregate on each timestamp, rather than on the highest or lowest timestamp.
Example:
id:xyz,type:polar,interval:day_of_week
"xyz" : [
  { "value" : 1, "label" : "Mon", "count" : 26 },
  { "value" : 2, "label" : "Tue", "count" : 14 },
  { "value" : 3, "label" : "Wed", "count" : 10 },
  ...
]

Errors

The following error codes could be returned for most requests:

400 Bad Request
Invalid or missing parameters or request body; check the response body for an explanation.
401 Unauthorized
Missing Authentication header.
403 Forbidden
Authentication failed, or insufficient permissions.
404 Not Found
The requested resource doesn't exist.
409 Conflict
Trying to update a stale resource.
500 Internal Server Error
Shouldn't happen; please contact us.
503 Service Unavailable
The site is down for maintenance or overloaded; contact us if this lasts longer than a few minutes.

The response may contain a body with a description of the problem:

{
  "message" : <String>
}
